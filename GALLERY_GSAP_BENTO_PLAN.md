# 美图鉴赏页 GSAP Scrubbed Bento 改造方案

## 1. 目标

把当前 `/gallery` 页面改造成「滚动驱动的 Scrubbed Bento Gallery」：

- 中间主视觉用 bento grid 呈现，强调大图 + 小图穿插。
- 滚动时用 GSAP + ScrollTrigger 做 scrub 联动，而不是单纯的进入动画。
- 右侧信息栏保留，作为“作品说明 / 作者 / 标签 / 操作”面板。
- 先保留现有图片数据与 lightbox 能力，优先替换展示层。

## 2. 现状判断

当前页面已经具备几个适合复用的基础：

- 路由已存在：`/gallery`
- 数据源已整理：`src/data/gallerySources.js`、`src/data/officialGallerySources.js`
- 图片资源已统一放在 `public/gallery`
- 项目已安装 `gsap`

所以这次改造的重点不是“重做数据”，而是“重做版式与滚动编排”。

## 3. 推荐实现思路

### 3.1 页面结构

建议把页面拆成 3 层：

1. 左侧导航栏
2. 中间 gallery 舞台
3. 右侧作品信息栏

中间舞台再拆为：

- `sticky` 容器
- `bento grid`
- `scroll progress` 驱动层
- `lightbox` 层

### 3.2 视觉策略

建议采用“深色底 + 高对比内容块 + 局部发光”的方向：

- 页面背景：接近黑色，带轻微噪点或渐变
- 卡片：深灰半透明，边缘圆角统一
- 主图：大面积满裁切，少量留白
- 点缀：用 1 个强调色控制按钮、进度线、当前激活态

### 3.3 动画策略

GSAP 只负责“滚动和状态联动”，不要把所有细节都塞进动画里：

- `ScrollTrigger` 驱动整体 scrub
- `pin` 固定主舞台
- `timeline` 控制卡片缩放、位移、透明度、模糊度
- 当前聚焦卡片随滚动切换
- 右侧面板可跟随当前激活项更新文案

推荐动画顺序：

1. 页面首屏进入，标题和主卡片轻入场
2. 滚动时 bento 网格逐步重排
3. 中央主视觉从大图切到下一组图
4. 右侧信息同步切换

## 4. 数据组织建议

建议不要把动画和数据混在同一个文件里，拆成三层更稳：

- `gallery data`：图片、标题、来源、标签
- `gallery layout config`：每个卡片占几格、初始权重、激活顺序
- `gallery animation config`：每段滚动对应的位移、缩放、时长

这样后面你想换主题、换图片、换动画节奏，改动会比较小。

## 5. 分阶段实施

### Phase 1

只改布局，不上复杂动画：

- 做出 bento grid 结构
- 固定右侧信息栏
- 接通现有图片和 lightbox

### Phase 2

加入 GSAP scrub：

- ScrollTrigger pin 主舞台
- 卡片按滚动节奏缩放和位移
- 当前激活作品跟随滚动切换

### Phase 3

补体验细节：

- hover 放大和描边高亮
- 图片加载骨架
- 移动端降级为横向卡片流或单列流
- `prefers-reduced-motion` 降级处理

## 6. 实施建议

- 保留现有图片数据，不重建资源体系。
- 先做一个“概念版 gallery section”，再迁移到正式页。
- GSAP 动画只绑定 DOM 引用，不要把业务状态写死在 timeline 里。
- 右侧信息栏建议保持半固定，避免滚动时视觉太乱。
- 移动端不要强行复刻桌面 bento，优先可读性。

## 7. 风险点

- 图片过多时会导致首屏渲染压力大，建议做分组或懒加载。
- scrub 动画过密会让页面显得“晕”，需要控制节奏。
- 如果 lightbox 和滚动动画同时强交互，要注意层级和滚动锁定。

## 8. 验收标准

- 页面视觉接近示意图的“bento + 深色 + 侧栏”风格
- 滚动时有明确的 scrub 联动感
- 现有图片点击预览仍可用
- 桌面端和移动端都能正常浏览
- `prefers-reduced-motion` 下不会破坏可读性
