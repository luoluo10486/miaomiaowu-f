# 美图鉴赏页 GSAP Scrubbed Bento 改造方案

## 1. 目标

把当前 `/gallery` 页面改造成「滚动驱动的 Scrubbed Bento Gallery」：

- 中间主视觉用 bento grid 呈现，强调大图 + 小图穿插。
- 先点击选中某一张图，再在后续滚动中用 GSAP + ScrollTrigger 做 scrub 联动。
- 右侧信息栏保留，作为“作品说明 / 作者 / 标签 / 操作”面板。
- 先保留现有图片数据与 lightbox 能力，优先替换展示层。

## 1.1 核心交互

这次改造的核心不是“用户一进页面就开始滚动动画”，而是下面这个顺序：

1. 用户先点击一张图
2. 该图进入“选中态”
3. 页面视觉重心切到这张图
4. 用户继续向下滚动
5. 从这里开始，GSAP scrub 才驱动放大、位移、重排、信息联动

也就是说：

- `click` 负责“选中目标”
- `scroll` 负责“驱动演出”

这是和你想要的效果最一致的交互模型。

## 2. 现状判断

当前页面已经具备几个适合复用的基础：

- 路由已存在：`/gallery`
- 数据源已整理：`src/data/gallerySources.js`、`src/data/officialGallerySources.js`
- 图片资源已统一放在 `public/gallery`
- 项目已安装 `gsap`

所以这次改造的重点不是“重做数据”，而是“重做版式与滚动编排”。

## 3. 推荐实现思路

### 3.1 页面结构

建议把页面拆成 3 层：

1. 左侧导航栏
2. 中间 gallery 舞台
3. 右侧作品信息栏

中间舞台再拆为：

- `sticky` 容器
- `bento grid`
- `scroll progress` 驱动层
- `lightbox` 层

建议再补 2 个状态层：

- `selection state`：当前被点击选中的图片
- `scrub stage state`：当前滚动演出进行到哪一段

### 3.2 视觉策略

建议采用“深色底 + 高对比内容块 + 局部发光”的方向：

- 页面背景：接近黑色，带轻微噪点或渐变
- 卡片：深灰半透明，边缘圆角统一
- 主图：大面积满裁切，少量留白
- 点缀：用 1 个强调色控制按钮、进度线、当前激活态

### 3.3 动画策略

GSAP 只负责“滚动和状态联动”，不要把所有细节都塞进动画里：

- `ScrollTrigger` 驱动整体 scrub
- `pin` 固定主舞台
- `timeline` 控制卡片缩放、位移、透明度、模糊度
- 当前聚焦卡片优先由点击选中决定
- 右侧面板可跟随当前激活项更新文案

推荐动画顺序：

1. 页面首屏进入，标题和 gallery 卡片轻入场
2. 用户点击某张图，进入选中态
3. 选中图在本地做一次轻量聚焦反馈
4. 用户开始滚动后，bento 网格进入 scrub 阶段
5. 选中图逐步放大、居中、压住周围卡片
6. 周围卡片按时间轴退让、错位、裁切或淡出
7. 右侧信息栏同步展示当前选中作品

## 3.4 交互状态机建议

建议不要把所有逻辑塞在一个布尔值里，而是拆成明确状态：

- `idle`：未选中，页面只是普通 gallery
- `selected`：已点击某张图，但尚未进入 scrub 滚动段
- `scrubbing`：已进入滚动驱动阶段
- `settled`：滚动结束，选中图完成主视觉演出

对应行为建议：

- `idle -> selected`
  由点击图片触发
- `selected -> scrubbing`
  由滚动进入指定 trigger 区域触发
- `scrubbing -> settled`
  由时间轴结束或滚动到尾段触发
- `selected -> idle`
  由点击空白区域、点关闭、切换选中项触发

## 4. 数据组织建议

建议不要把动画和数据混在同一个文件里，拆成三层更稳：

- `gallery data`：图片、标题、来源、标签
- `gallery layout config`：每个卡片占几格、初始权重、激活顺序
- `gallery animation config`：每段滚动对应的位移、缩放、时长

另外再补一层交互状态：

- `gallery interaction state`：`selectedId`、`isScrubbing`、`activeStage`

这样后面你想换主题、换图片、换动画节奏，改动会比较小。

## 5. 分阶段实施

### Phase 1

只改布局，不上复杂动画：

- 做出 bento grid 结构
- 固定右侧信息栏
- 接通现有图片数据
- 改成“点击选中”而不是“点击直接进入演出”

### Phase 2

加入 GSAP scrub：

- ScrollTrigger pin 主舞台
- 只有在存在 `selected` 图片时才启动 scrub 时间轴
- 选中图片按滚动节奏缩放、位移、放大
- 周围卡片围绕选中图做退让和重排
- 右侧信息栏绑定当前选中项，而不是随机切换

### Phase 3

补体验细节：

- hover 放大和描边高亮
- 选中态边框、阴影、标题强化
- 图片加载骨架
- 移动端降级为横向卡片流或单列流
- `prefers-reduced-motion` 降级处理

## 6. 实施建议

- 保留现有图片数据，不重建资源体系。
- 先做一个“可点击选中”的 gallery 概念版，再迁移到正式页。
- GSAP timeline 不要默认驱动第一张图，必须绑定当前 `selected` 项。
- 点击时先做轻反馈，不要立刻触发整段重动画。
- 右侧信息栏建议绑定选中图，避免滚动时内容失焦。
- 移动端不要强行复刻桌面 bento，优先可读性。

## 7. 与现有 lightbox 的关系

这个需求会改变当前“点击图片”的默认语义，所以要提前定规则。

建议二选一：

1. 单击选中，右侧按钮打开预览
2. 单击选中，双击打开 lightbox

更推荐第 1 种，因为它和“点击后进入滚动演出”不会冲突。

## 8. 风险点

- 图片过多时会导致首屏渲染压力大，建议做分组或懒加载。
- scrub 动画过密会让页面显得“晕”，需要控制节奏。
- 如果 lightbox 仍然占用单击事件，就会和“点击选中”冲突。
- 如果切换选中项时上一个 timeline 没有正确销毁，容易出现错位。

## 9. 验收标准

- 页面视觉接近示意图的“bento + 深色 + 侧栏”风格
- 点击图片后会明确进入选中态
- 只有在选中后滚动，才会出现 scrub 联动感
- 选中图的放大、聚焦、重排逻辑明确
- 图片预览能力仍可用，但不和“单击选中”冲突
- 桌面端和移动端都能正常浏览
- `prefers-reduced-motion` 下不会破坏可读性

## 10. 推荐下一步

如果要继续，我建议下一步直接做：

1. 先输出 `GalleryView.vue` 的“点击选中 + 滚动 scrub”结构草图
2. 再拆 `selected / scrubbing / settled` 这几个状态
3. 最后再接 GSAP timeline
